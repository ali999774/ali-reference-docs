🛡 exec Command Allowlist

shell safety tiers · ai agent exec wrappers

Wrapper Policy

default deny-all; only allowlisted commands pass through exec()
scope per-command AND per-flag — a safe command with a dangerous flag is blocked
cwd constrained to workspace root; path traversal (../) is rejected pre-exec
env stripped to minimal set; no credential env vars forwarded
timeout hard 30s wall-clock per exec call; long-running blocked at spawn
pipes stdout/stderr captured; shell operators (|, &, ;, `) require explicit approval
T1 · Safe (read-only)
T2 · Allowed (reversible write)
T3 · Restricted (confirm required)
T4 · Network (audit logged)
T5 · Blocked (never exec'd)
T1 · Safe

Read-Only — exec without confirmation

No side effects · idempotent
ls / dir
List directory contents.
-la-R--color
cat / head / tail
Print file contents. tail -f blocked (blocking I/O).
-n-f (tail)
pwd
Print working directory. Always safe.
echo / printf
Print to stdout. Redirect to file upgrades to T2.
stdout> file
grep / rg / ag
Search file contents. Pattern match only.
-r-i-n-l
find
Locate files by name/type/date. -exec upgrades tier.
-name-type-exec rm
wc / sort / uniq
Count, sort, deduplicate. Pure stream transforms.
diff / diff3
Compare file contents. Patch application is T2.
-u-r
stat / file / du
File metadata, type detection, disk usage.
-h-s
env / printenv
Print environment. Filtered — credentials stripped before output.
which / type / whereis
Locate binaries. Safe lookup only.
date / uname / uptime
System info — time, OS version, load. Read-only.
T2 · Allowed

Reversible Writes — exec in workspace scope

Changes files · recoverable via git
mkdir
Create directories within workspace. Parent traversal blocked.
-p/ (root)
touch
Create empty file or update timestamp. Workspace-scoped.
cp
Copy file or directory. Source and dest must be within workspace.
-r-p
mv
Move or rename. Both paths workspace-scoped. Overwrite allowed.
echo / printf > file
Write stdout to file within workspace. Redirect operator requires T2.
sed -i / awk
In-place text transforms on workspace files. Backup flag recommended.
-i.bak-E
patch
Apply unified diff. Files must be within workspace.
-p1--dry-run
git (read/stage)
status, log, diff, add, commit within repo. Push/remote = T3.
statuslogdiffaddcommit
zip / tar (create)
Archive workspace files. Extraction to workspace only.
-czf-C /
node / python / ruby
Run local scripts. Network calls from script = T4. System calls = T3.
--no-warnings-e 'import os'
T3 · Restricted

Confirm Required — human approval before exec

Broader system impact · may be irreversible
rm
Delete files. Always requires confirmation. -rf on directories double-confirms.
-rf--no-preserve-root
chmod / chown
Change permissions or ownership. Confirm for recursive or 777 patterns.
-R777
git push / pull / clone
Remote git operations. Requires confirm + credential scope check.
--dry-run--force
npm / pip / brew install
Package installs modify system or project deps. Confirm + lockfile review.
--dry-run-g (global)
make / cmake / gradle
Build systems can exec arbitrary targets. Confirm target name.
-n (dry)installdeploy
docker run / compose
Containers with volume mounts, port binding, or privileged mode.
--privileged-v /:/
systemctl / launchctl
Start/stop system services. Confirm service name and action.
enabledisable
crontab / at
Schedule future commands. Requires explicit approval + content review.
-e-l (list)
T4 · Network

Audit Logged — all network exec is recorded

URL + payload logged · confirm + scope check
curl
HTTP requests. URL allowlist checked. Pipe to shell blocked at parse time.
-I (HEAD)-o file| sh| bash
wget
File download. Destination must be workspace. Recursive crawl blocked.
-O outfile-r--mirror
ssh
Remote shell. Host must be in allowlist. Agent forwarding blocked.
-A (fwd)-L (tunnel)
scp / rsync
Remote copy. Both endpoints verified. No wildcard remote paths.
-n (dry)--delete
ping / traceroute / nslookup
Network diagnostics. Read-only but network egress — logged.
-c 4
gh / git remote
GitHub CLI and remote git ops. Credentials scoped, actions logged.
pr listissue viewrepo delete
T5 · Blocked

Never Exec'd — rejected before process spawn

Denied regardless of framing or context
sudo / su / doas
Privilege escalation. Blocked unconditionally. No exceptions.
rm -rf /
Root deletion. Rejected by path check before any other evaluation.
dd
Raw disk I/O. Can overwrite block devices. Always blocked.
if=/dev/zeroof=/dev/sd*
mkfs / fdisk / parted
Partition and format operations. Irreversible data loss risk.
kill -9 / killall
Force-kill processes. Blocked to prevent host disruption or self-termination.
-9-KILL
curl … | sh
Remote code execution pattern. Detected at AST parse. Always denied.
nc / netcat (listen)
Bind sockets / reverse shells. -l flag triggers immediate block.
-l-e /bin/sh
iptables / pfctl / ufw
Firewall modification. Network policy changes always blocked.
crontab -r
Delete all cron jobs. Destructive and non-workspace. Blocked.
history -c / shred
Evidence destruction. Blocked for audit integrity.

Implementation Notes

parse firstTokenize the full command string before exec. Flag conflicts (safe cmd + dangerous flag) are caught here, not at runtime.
no shell=TrueUse execvp() / subprocess list form — never shell=True. Shell operators bypass the allowlist entirely when shell=True is set.
chroot / sandboxRun T2–T3 in a chroot or container with workspace bind-mounted read-write and / read-only.
symlink checkResolve symlinks before path scope check. A symlink inside workspace pointing to /etc/passwd is a T5 attempt.
prompt injectionAgent input from untrusted sources (web pages, emails) must be sanitized before reaching exec wrapper — treat as hostile by default.
log formatEach exec: timestamp · tier · full argv · cwd · exit code · stdout hash. Stored append-only.
